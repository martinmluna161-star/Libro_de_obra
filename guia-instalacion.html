1
Google Apps Script Se hace una sola vez

Apps Script es el backend del sistema. Recibe los registros desde los HTMLs y los guarda en Google Sheets. También los sirve de vuelta cuando el reporte cliente o la base de datos los piden.

1
Crear el proyecto
Abrí script.google.com en tu navegador. Hacé clic en "Nuevo proyecto". Nombralo Diario de Obra — Sanchez Gil.
2
Pegar el código
Seleccioná todo el texto en el editor (Ctrl+A) y borralo. Abrí el archivo Code.gs que descargaste, copiá todo su contenido y pegalo en el editor. Guardá con Ctrl+S.
💡
Opcional: Si ya tenés un Google Sheets existente donde querés guardar los datos, copiá el ID de su URL (docs.google.com/spreadsheets/d/ESTE-ID/edit) y pegalo en la línea const SHEET_ID = 'PEGAR-AQUI'. Si lo dejás vacío, el sistema crea el Sheets automáticamente.
3
Implementar como Web App
En el menú superior hacé clic en Implementar → Nueva implementación. Luego:
CampoValor a seleccionar
TipoHacé clic en ⚙ → seleccioná Aplicación web
Descripciónv1.0 (cualquier texto)
Ejecutar comoYo (tu cuenta de Google)
Quién tiene accesoCualquier persona
Hacé clic en Implementar. Google va a pedirte que autorices permisos de Google Sheets y Drive — hacé clic en Autorizar acceso y aceptá todo.
4
Copiar la URL generada ← IMPORTANTE
Después de implementar, Apps Script muestra una URL de aplicación web. Copiá esa URL y guardala — la vas a usar en el Paso 3.
Formato de la URL
https://script.google.com/macros/s/AKfycb.../exec
⚠
Guardá esta URL en un lugar accesible (un notepad, un mail a vos mismo). Si la perdés podés recuperarla desde Implementar → Administrar implementaciones.

Conectar la URL del Apps Script Se hace una sola vez

Los HTMLs necesitan saber dónde está el Apps Script. La configuración se guarda en el navegador, así que solo hay que hacerlo una vez por dispositivo.

📝 libro-diario-obra.html
  1. Abrí el archivo desde tu URL de GitHub Pages
  2. Andá a la pestaña ⚙ Config
  3. Pegá la URL del Apps Script en el campo
  4. Hacé clic en "Probar conexión"
  5. Tiene que aparecer ✅
📊 base-datos-obra.html
  1. Abrí el archivo desde GitHub Pages
  2. En el banner oscuro del tope, pegá la URL
  3. Hacé clic en "Conectar y cargar datos"
  4. El indicador cambia a verde: ● Sheets
💡
reporte-cliente.html no necesita configuración manual — lee la URL automáticamente desde el mismo navegador una vez que la configuraste en el libro.
⚠
La configuración queda guardada por navegador y por PC. Si abrís los archivos en otra computadora, tenés que pegar la URL nuevamente. Es una operación de 10 segundos.

Solución de problemas
"No se pudo conectar con Sheets" al guardar
1. Verificá que la URL del Apps Script termine en /exec (no en /dev).
2. Confirmá que la Web App tiene acceso configurado como "Cualquier persona" en Apps Script → Implementar → Administrar.
3. Si modificaste el código, ¿creaste una nueva versión al reimplementar? (no basta con guardar el código).
4. El dato llega a Sheets igual — el fetch modo no-cors no devuelve respuesta pero el POST se ejecuta. Verificá en el Sheets directamente.
El Sheets no se creó automáticamente
1. Verificá que autorizaste los permisos de Google Drive y Sheets cuando te lo pidió al implementar.
2. Abrí Apps Script → menú Ejecutar → getSpreadsheet manualmente para que cree el archivo.
3. Revisá la carpeta "Mi unidad" y también "Compartido conmigo" en Drive.
El reporte del cliente muestra "Sin registros"
1. El nombre del proyecto en la URL tiene que ser idéntico al cargado en el libro (mayúsculas, espacios, acentos).
2. Usá la función "Copiar link" de la pestaña Config del libro para evitar errores de tipeo.
3. Verificá que la URL del Apps Script esté configurada en el navegador del cliente (o que el sistema la lea desde localStorage).
GitHub Pages muestra error 404
1. Verificá que Pages esté activado en Settings → Pages.
2. Confirmá que los archivos están en la rama main (no master).
3. Esperá hasta 5 minutos después del primer deploy — a veces tarda.
4. En Settings → Pages debería aparecer un banner verde con la URL activa.
Apps Script pide autorización cada vez
Esto pasa si la cuenta cambió o si los permisos expiraron. Abrí Apps Script → Implementar → Administrar implementaciones → editá → creá nueva versión → reautorizá. La URL del endpoint no cambia.
Los datos se ven en caché pero no carga desde Sheets
El sistema funciona en modo offline automáticamente — muestra los últimos datos cacheados. Para forzar la recarga desde Sheets, hacé clic en ↻ Actualizar en base-datos-obra.html y verificá que tenés conexión a internet y la URL configurada correctamente.
